# 2026-03-31 ENA E2E生成中断再生成{main}

## 目的
- EasyNovelAssistant から `http://127.0.0.1:5001` 経由で `生成 -> 中断 -> 再生成` の運用可否を確認。

## 実施環境
- Hypura: `http://127.0.0.1:8101`
- Proxy: `http://127.0.0.1:5001` (`kobold_to_hypura_proxy.py`)
- Model: `Qwen3.5-9B-Uncensored-HauhauCS-Aggressive-Q4_K_M.gguf`

## 事前確認
- `GET /api/v1/model` via proxy: 200
- `POST /api/v1/generate` via proxy: 200

## 実施内容（3サイクル）
- 各サイクル:
  1. 長め prompt で `POST /api/v1/generate`
  2. 0.8s 後に `POST /api/extra/abort`
  3. `GET /api/extra/generate/check`
  4. 短文 prompt で `POST /api/v1/generate`（再生成）

## 結果
- 3サイクルすべてで:
  - abort: 200
  - check: 200
  - 再生成: 200
- ただし、`/api/v1/generate` は同期レスポンスのため、abort実行時点で長文生成が完了しているケースが多く、「途中停止」の見かけ効果は限定的。

## 切り分け（失敗/制約）
- 主因: proxy の `/api/v1/generate` が同期フォワード（`/api/generate`）で、UI側の abort と時間競合する。
- 対応: proxy に `/api/extra/abort` と `/api/extra/generate/check` の Hypura フォワードを追加済み。
- 残課題: 真の途中停止体験を強化するには、UI側を stream 主体へ寄せるか、proxy 側で非同期化を導入する。

## 追補（stream主体化トライ）
- proxy の `/api/v1/generate` を `/api/extra/generate/stream` 利用へ切替。
- 生成進捗を共有状態に載せて `/api/extra/generate/check` で返す実装を追加。
- ただし今回環境では check 応答が空のままになるケースが継続し、体感改善は限定的。
- abort/retry は 200 応答で成立するが、「途中停止中の部分出力可視化」は追加調査が必要。

## 追補2（stream受信化の最小差分実装）
- `EasyNovelAssistant/src/kobold_cpp.py`
  - `generate_stream()` を追加し、Hypura `/api/extra/generate/stream` を逐次受信。
  - `abort()` は backend 分岐を外し、常に `POST /api/extra/abort` を実行。
- `EasyNovelAssistant/src/generator.py`
  - stream callback で token をバッファリングし、`update()` で段階描画。
  - Hypura backend 時は check ポーリング依存を弱め、stream描画を優先。
  - `abort()` 時に stream バッファをクリアし、生成ジョブをキャンセル。
- `proxy/kobold_to_hypura_proxy.py`
  - `/api/v1/generate` は stream 受信ベースで互換レスポンスを返却。
  - `/api/extra/generate/check` は `{"results":[{"text":...}], "in_progress": ...}` を返却。

## 追補2 検証（2026-03-31）

- E2E 3サイクル:
  - 各サイクルで `partial=True`, `abort=200`, `first=200`, `retry=200`。
